mock_market 操作手册

mock_market 是 StockSim 的模拟公司子系统，提供 5-10 家24 小时开盘的模拟股票，价格数据持久化到独立数据库，支持通过 API 实时调节超参数。

关键特性：

  • 独立数据库：mock_market.db 与用户/交易数据（data/sql_app.db）完全隔离，删除不影响用户账号
  • 24/7 运行：不受 A 股交易时段限制，以 UTC 日期为模拟日，凌晨 0 点自动切换
  • 崩溃恢复：引擎每 10 个 tick 写一次快照，重启后恢复状态并快进缺失的天数
  • 实时调参：通过管理 API 修改超参数，下一个 tick 立即生效（日收益率类参数在次日生效）

快速开始

启动服务器

python main.py

首次启动时自动完成：

  1. 创建 mock_market.db 数据库文件
  2. 播种 10 家默认模拟公司（代码 m00001 ~ m00010）
  3. 为每家公司生成 2020-01-01 至今的全部历史日线
  4. 启动后台价格引擎，开始实时生成 tick

查看模拟公司

在浏览器中打开 http://127.0.0.1:7999，右侧边栏会显示 "🏭 模拟公司" 区域，列出所有公司的快捷按钮。点击即可查看 K 线图。

查看公司配置

curl http://127.0.0.1:7999/api/admin/companies

价格模型

价格的生成分为两个层次：日收益率（决定每天的走向）和日内 tick（决定盘中价格路径）。

日收益率模型

每天凌晨（UTC），引擎为新一天生成一个对数收益率：

daily_return ~ N(μ, σ²)
close_target = open × exp(daily_return)

其中 μ 是 daily_drift_mu，σ 是 daily_sigma。

这等价于经典的几何布朗运动（Geometric Brownian Motion），μ 控制长期趋势，σ 控制波动幅度。

日内 Tick 模型（布朗桥）

每个模拟日被分成 ticks_per_day 个 tick。单个 tick 的价格变动为：

progress = tick_index / ticks_per_day
target   = day_open + (day_close_target − day_open) × progress
reversion = (target − price) × mean_reversion
noise    = price × N(0, tick_sigma²)

price_new = price + reversion + noise

这被称为布朗桥（Brownian Bridge）——价格在日内从开盘价走向目标收盘价，同时：

  • 均值回归（mean_reversion）将价格拉回"桥"的路径上
  • 随机噪声（tick_sigma）让价格不断抖动，模拟真实市场的微观波动

最终钳制在 [price_min, price_max] 区间内。

超参数详解

下面按影响层级逐一解释每个超参数。

initial_price — 初始价格

属性 值
类型 float
默认值 50.00
范围 0.01 ~ 100000.0
生效时机 创建公司时 / 重置时

公司诞生时的价格。在首次创建或执行 reset 操作后，价格从该值开始运行。已运行的公司修改此参数不会影响当前价格（除非 reset）。

调参建议：建议设在 10~200 之间，与 A 股常识接轨。极低值（< 1）会放大百分比噪声的视觉效果；极高值（> 10000）会使 K 线图难以阅读。

daily_drift_mu — 日均对数收益率（漂移）

属性 值
类型 float
默认值 0.0002
范围 -0.05 ~ 0.05
生效时机 次日

这是决定股票长期走势最重要的参数。

μ 是日对数收益率的均值，决定了价格的长期趋势：

μ 值 年化收益率（约） 效果
0.0005 ~13% 强势上涨
0.0002 ~5% 温和上涨（默认）
0.0000 ~0% 横盘震荡
-0.0001 ~-2.5% 缓慢阴跌
-0.0005 ~-12% 持续下跌

调参建议：模拟"成长股"可设 0.0004~0.0008；"蓝筹股"设 0.0001~0.0002；"夕阳行业"可设为负值。注意该值在 UTC 凌晨切日时作用，修改后次日才生效。

daily_sigma — 日波动率

属性 值
类型 float
默认值 0.02
范围 0.001 ~ 0.20
生效时机 次日

这是决定股票波动剧烈程度最重要的参数。

σ 是日对数收益率的标准差。由于日收益率遵循 N(μ, σ²)，σ 越大，每天涨跌幅的不确定性越大：

σ 值 年化波动率（约） 日涨跌幅集中区间 效果
0.005 ~8% ±0.5% 极低波动，像货币基金
0.010 ~16% ±1% 低波动，像大盘蓝筹
0.020 ~32% ±2% 中等波动（默认）
0.035 ~56% ±3.5% 高波动，像小盘成长股
0.050 ~80% ±5% 极高波动，像加密货币
0.100 ~160% ±10% 极端波动，一天可能翻倍/腰斩

调参建议：10 家默认公司已预设了不同 σ（0.015 ~ 0.030），可以此作为参考。想让某只股票更"刺激"就把 σ 调大，但要配合 price_min / price_max 防止价格归零或爆炸。

mean_reversion — 均值回归强度

属性 值
类型 float
默认值 0.06
范围 0.0 ~ 1.0
生效时机 立即（下一个 tick）

控制价格在日内被拉回布朗桥路径的力度。

reversion = (target − price) × mean_reversion
  • 0.0：完全无回归，价格做纯随机游走，收盘可能偏离目标很远
  • 0.03~0.08：温和回归（默认区间），价格在目标附近波动
  • 0.2~0.5：强回归，价格紧贴布朗桥路径，日内走势非常平滑
  • 1.0：每个 tick 都几乎完全回归到目标位置，几乎没有随机性

调参建议：默认 0.06 产生的走势比较自然。如果想让日内波动更"野"，降到 0.02~0.03；如果希望走势非常规整、较少意外，可以提高到 0.1~0.15。

tick_sigma — 单 Tick 噪声标准差

属性 值
类型 float
默认值 0.003
范围 0.0 ~ 0.10
生效时机 立即（下一个 tick）

单个 tick 的随机噪声强度：

noise = price × N(0, tick_sigma²)
tick_sigma 单 tick 典型跳幅 视觉效果
0.0 0% 无噪声，纯布朗桥平滑曲线
0.001 ~0.1% 几乎看不出来，非常平滑
0.003 ~0.3% 自然的微波动（默认）
0.010 ~1% 明显抖动，每一帧都在跳
0.030 ~3% 剧烈抖动，像高频交易

调参建议：这是"微观"参数，影响的是 WebSocket 实时推送中价格的变化幅度。tick_sigma 和 tick_interval_seconds 需要配合使用——间隔越短，单 tick 噪声应该越小，否则看起来会非常抖。

tick_interval_seconds — Tick 间隔（秒）

属性 值
类型 int
默认值 30
范围 1 ~ 3600
生效时机 立即（下一个 tick）

真实时间中两个 tick 之间的秒数。

  • 1~5 秒：极快，适合 WebSocket 演示时展示流畅的价格动画
  • 10~30 秒：适中（默认 10 家公司的范围是 20~40 秒）
  • 60~300 秒：慢速，接近真实分钟级行情
  • 3600 秒：每 tick 一小时，几乎无实时感

调参建议：演示时建议 2~5 秒；正常运行时建议 20~60 秒。配合 ticks_per_day 可以控制一个模拟日的总时长：总时长 = tick_interval_seconds × ticks_per_day ÷ 86400 天。

ticks_per_day — 每日 Tick 数

属性 值
类型 int
默认值 240
范围 10 ~ 1440
生效时机 次日

一个模拟日中包含的 tick 总数。决定了日内价格路径的精细程度。

  • 10：极粗糙，一天只有 10 个点，K 线信息量少
  • 60~120：粗糙，适合不关心日内细节的场景
  • 240：默认值，相当于每小时 10 tick，日内走势比较完整
  • 1440：每分钟一个 tick，日内走势非常细致

调参建议：默认 240 足够了。如果 tick_interval_seconds = 30 且 ticks_per_day = 240，则一个模拟日 = 240 × 30 = 7200 秒 = 2 小时。修改后次日生效。

price_min / price_max — 价格上下限

属性 值
类型 float
默认值 0.50 / 10000.0
范围 无硬性限制（建议合理设置）
生效时机 立即（下一个 tick）

价格的绝对钳制区间。每个 tick 生成后，价格被裁剪到此范围内：

if price < price_min: price = price_min
if price > price_max: price = price_max

这是安全网——防止 daily_drift_mu 为负且 daily_sigma 很大时价格归零，也防止价格无限增长。

调参建议：默认值对绝大多数场景足够安全。如果模拟低价"仙股"，可设 price_min = 0.01；如果模拟高价股（如伯克希尔），可设 price_max = 100000.0。

is_active — 是否活跃

属性 值
类型 bool
默认值 True
生效时机 立即

控制引擎是否继续为该代码生成价格。

  • true：正常生成 tick
  • false：停用（软删除），引擎跳过该公司

通过 DELETE /api/admin/companies/{code} 可将其设为 false。历史日线不会删除。重新激活只需 PUT 设置 is_active: true。

管理 API 参考

所有 API 均以 /api/admin 为前缀，无需认证（本地/开发用途）。

列出所有公司

GET /api/admin/companies

返回示例：

{
  "companies": [
    {
      "id": 1,
      "code": "m00001",
      "name": "星辰科技",
      "initial_price": 68.0,
      "daily_drift_mu": 0.0003,
      "daily_sigma": 0.018,
      "mean_reversion": 0.05,
      "tick_sigma": 0.0025,
      "tick_interval_seconds": 30,
      "ticks_per_day": 240,
      "price_min": 0.5,
      "price_max": 10000.0,
      "is_active": true,
      "created_at": "2026-06-25T12:00:00",
      "updated_at": "2026-06-25T12:00:00"
    }
  ],
  "total": 10
}

获取单家公司

GET /api/admin/companies/{code}

创建新公司

POST /api/admin/companies
Content-Type: application/json

{
  "code": "m00011",
  "name": "火箭科技",
  "initial_price": 100.0,
  "daily_sigma": 0.04,
  "tick_interval_seconds": 15
}

必填字段：code（格式 m + 5 位数字）、name（1~50 字符）。其余超参数使用默认值。

创建后引擎立即开始为新公司生成价格（包括历史日线和实时 tick）。

更新超参数（部分更新）

PUT /api/admin/companies/{code}
Content-Type: application/json

{
  "daily_sigma": 0.05,
  "tick_interval_seconds": 10
}

只需传要修改的字段，其余保持不变。日收益率类参数（daily_drift_mu、daily_sigma、ticks_per_day）次日生效，其余参数立即生效。

停用公司（软删除）

DELETE /api/admin/companies/{code}

将 is_active 设为 false，引擎停止生成 tick。历史数据保留。可通过 PUT 重新激活。

重置公司

POST /api/admin/companies/{code}/reset

效果：

  1. 删除该公司所有历史日线
  2. 价格回到 initial_price
  3. 重新生成 2020-01-01 至今的历史日线
  4. 从当天开始重新生成 tick

这是不可逆操作，用于彻底重来。

常见操作示例

让某只股票波动更剧烈

curl -X PUT http://127.0.0.1:7999/api/admin/companies/m00003 \
  -H "Content-Type: application/json" \
  -d '{"daily_sigma": 0.06, "tick_sigma": 0.008}'

让某只股票持续上涨

curl -X PUT http://127.0.0.1:7999/api/admin/companies/m00008 \
  -H "Content-Type: application/json" \
  -d '{"daily_drift_mu": 0.0008}'

创建一只"妖股"（极端波动 + 极快 tick）

curl -X POST http://127.0.0.1:7999/api/admin/companies \
  -H "Content-Type: application/json" \
  -d '{
    "code": "m00011",
    "name": "妖股科技",
    "initial_price": 50.0,
    "daily_drift_mu": 0.0000,
    "daily_sigma": 0.10,
    "mean_reversion": 0.02,
    "tick_sigma": 0.02,
    "tick_interval_seconds": 2,
    "ticks_per_day": 500
  }'

让 WebSocket 推送更流畅（更快的 tick）

curl -X PUT http://127.0.0.1:7999/api/admin/companies/m00001 \
  -H "Content-Type: application/json" \
  -d '{"tick_interval_seconds": 2}'

重置某只股票，从头开始

curl -X POST http://127.0.0.1:7999/api/admin/companies/m00006/reset

让价格走势更平滑（减少噪声）

curl -X PUT http://127.0.0.1:7999/api/admin/companies/m00004 \
  -H "Content-Type: application/json" \
  -d '{"mean_reversion": 0.15, "tick_sigma": 0.001}'

故障排查

服务器重启后数据还在吗？

在。 所有数据存储在 mock_market.db 中。重启后：

  • 日线历史从数据库加载
  • 引擎状态从 ticker_snapshots 表恢复
  • 停机期间流逝的天数会被自动快进（生成对应日线并入库）

如何完全清空模拟数据重来？

# 停止服务器，删除数据库文件
rm mock_market.db mock_market.db-wal mock_market.db-shm

# 重新启动，会自动重建
python main.py

价格不动了怎么办？

检查：

  1. is_active 是否为 true：GET /api/admin/companies/{code}
  2. 引擎是否崩溃（查看终端日志，应有 [MockEngine] 前缀的输出）
  3. tick_interval_seconds 是否设得过大

并发访问是否有问题？

引擎内部使用 threading.Lock 保护每家公司状态，WebSocket 读取和后台 tick 推进互不干扰。SQLite 使用 WAL 模式，允许多个读取操作与一个写入操作并发。

历史数据太多导致数据库过大？

10 家公司 × 每年约 365 条日线 = 每年约 3650 行。SQLite 可以轻松处理数万行数据。如果确实需要清理，可以删除特定公司的旧日线（目前无内置 API，可操作 SQLite）。
